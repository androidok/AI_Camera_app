# 相机画质调试指南

## 一、连接真机

### 1. 启用开发者选项
```
设置 → 关于手机 → 连续点击"版本号"7次
```

### 2. 开启USB调试
```
设置 → 开发者选项 → 开启"USB调试"
```

### 3. 连接电脑
用USB线连接手机和电脑，在手机上点击"允许USB调试"。

---

## 二、查看日志的方法

### 方法一：Android Studio Logcat（推荐）

1. 打开 Android Studio
2. 底部工具栏点击 **Logcat**
3. 连接手机后自动识别设备
4. 在过滤框输入标签名：
   - `QualityConfig` - 查看相机硬件能力和拍照配置
   - `CameraScreen` - 查看预览分辨率
   - `CameraBackend` - 查看拍照结果

### 方法二：ADB 命令行

#### 安装 ADB
下载地址：https://developer.android.com/tools/releases/platform-tools

#### 常用命令
```bash
# 查看连接的设备
adb devices

# 实时查看所有相机相关日志
adb logcat -s QualityConfig:D CameraScreen:D CameraBackend:D

# 只看分辨率相关日志
adb logcat | grep -E "分辨率|MP|画质|像素"

# 清除旧日志后重新查看
adb logcat -c && adb logcat -s QualityConfig:D CameraScreen:D CameraBackend:D

# 保存日志到文件
adb logcat -s QualityConfig:D CameraScreen:D CameraBackend:D > camera_debug.log
```

## 四、画质不如原相机的可能原因

### 1. 预览分辨率过低
- **现象**：预览模糊，但拍照清晰
- **原因**：CameraX 自动选择了较低的预览分辨率
- **解决**：在 Preview.Builder 中设置更高的目标分辨率

### 2. 图像处理差异
- **现象**：照片色彩、锐度不如原相机
- **原因**：原相机有厂商定制的图像处理算法
- **解决**：启用 HDR、调整锐化参数

### 3. JPEG 压缩质量
- **现象**：照片有压缩伪影
- **原因**：JPEG 质量设置
- **当前设置**：已设置为 100（最高）

### 4. 对焦和曝光
- **现象**：照片模糊或曝光不准
- **原因**：对焦模式、测光模式
- **当前设置**：连续对焦 + 自动抗频闪

### 5. 传感器读取方式
- **现象**：暗光下噪点多
- **原因**：原相机可能使用多帧合成
- **解决**：启用 HDR 模式

## 五、对比测试方法

### 1. 同场景对比
- 用本应用和原相机拍摄同一场景
- 对比日志中的分辨率数据
- 对比文件大小和实际像素

### 2. 检查关键指标
```
对比项：
├── 拍照分辨率是否达到硬件最高值
├── JPEG 质量是否为 100
├── 文件大小是否合理（12MP 照片通常 3-8MB）
└── 是否启用了 HDR 等增强模式
```
